# Distribution Channel Analysis: "今天整点啥" (Personal Hobby Project)

> 日期: 2026-06-13 | 项目阶段: 发现期 | 预算: 零 | 目标: 深圳年轻职场人(20-35岁)

---

## 核心原则

- **零预算**: 不做任何付费投放,不买量,不投广告
- **朋友优先**: 产品本质是"朋友间的协调工具",天然适合熟人传播
- **微信原生**: 目标用户微信依赖度极高,H5链接在微信内打开是唯一有意义的场景
- **轻量**: 不需要App Store审核,不需要小程序审核/备案,H5发链接就能用

---

## 零成本渠道分级(按影响力排序)

### Tier 1: 即刻启动(第1周)

#### 1.1 微信群直接分享(最核心渠道)

**为什么是第一渠道:**
- 目标用户(深圳职场人)每天在微信群里的时间超过任何其他场景
- 产品的使用场景天然是"群里有人发起约饭"——分享行为就是产品使用行为
- H5链接在微信内直接打开,无需跳转外部浏览器(微信内置浏览器兼容即可)

**分享卡片优化(这是技术关键点):**
- 必须接入微信JSSDK,调用`updateAppMessageShareData`和`updateTimelineShareData`
- 分享到群聊时显示自定义卡片:标题+描述+缩略图
- 卡片标题建议: "今天整点啥? [群名]的约饭协调中..."
- 卡片描述建议: "[发起人]邀请大家一起决定吃啥,3分钟搞定"
- 缩略图: 使用带有美食emoji或食物插画的品牌图
- JSSDK接入前提: 需要一个已认证的微信公众号(服务号),获取appId和appSecret
- **注意**: 未认证的公众号JSSDK权限受限;个人订阅号无法使用分享接口
- **折中方案**: 如果没有认证服务号,至少优化Open Graph标签(`og:title`,`og:description`,`og:image`),微信内置浏览器会读取这些标签生成基础卡片

**传播流程设计:**
```
发起人创建房间 → 获得"房间码"(4-6位数字) → 分享到微信群/好友
→ 朋友点击链接直接进入 → AI问偏好 → 等待决策结果
```

**房间码模式(借鉴微信接龙):**
- 短房间码(如"3951")比长URL更容易在群里传播
- 可以在分享文案中直接带上: "今天整点啥?房间码:3951,戳链接加入👉 [URL]"
- 房间码降低了"点击链接"的心理门槛——朋友知道这是一个轻量互动,不是下载App

#### 1.2 好友一对一转发

- 直接微信转发给2-3个最亲近的朋友做内测
- 一对一场景下转化率最高(朋友发来的东西大概率会点)
- 第一波种子用户应该是"操心人"型人格——他们天然愿意尝试新工具并传播

### Tier 2: 短期扩展(第1个月)

#### 2.1 朋友圈(Moments)分享

**优势:**
- 朋友圈是微信生态内第二大内容分发场景(仅次于群聊)
- 一次分享可触达所有微信好友,覆盖面广
- 适合发布"使用体验"类内容,而非硬广

**策略:**
- 在产品中加入"分享到朋友圈"按钮(调用JSSDK的`updateTimelineShareData`)
- 朋友圈分享文案设计: 展示产品截图+使用场景描述
  - 例: "试了下这个约饭工具,终于不用在群里反复问'吃啥'了...AI直接帮我们定了。房间码:3951,欢迎来试试😋"
- 不要过度刷屏——自然的、有信息量的分享比营销式发布效果好得多
- **朋友圈特性**: 隐私保护机制意味着只有共同好友才能看到彼此的点赞评论,无法形成公开讨论,但这也降低了分享者的社交压力

#### 2.2 小红书(RedNote/小红书)内容展示

- 小红书是当前中国年轻人发现新工具/新生活方式的核心平台
- **不做KOL投放**,而是自己发内容:
  - "深圳打工人约饭神器" - 使用场景展示
  - "朋友聚会再也不纠结吃啥了" - 痛点解决
  - "一个让AI帮我们决定午饭的周末实验" - 体验分享
- 内容形式: 截图+文字说明,展示产品核心流程(发起→投票→决策)
- 小红书图片要真实、生活化,避免过度设计的营销感
- 标签策略: #深圳美食 #打工人午餐 #朋友聚会 #AI工具 #效率工具

#### 2.3 即刻(Jike)社区分享

- 即刻是中文互联网产品/科技爱好者聚集地
- 适合发布产品思路和使用演示
- 即刻用户对"小而美的独立工具"接受度高
- 发帖风格: 坦诚分享开发过程和设计思考,而非宣传文案
- 可以加入"独立开发者"相关的即刻圈子

### Tier 3: 如有自然增长(第2个月起)

#### 3.1 少数派(sspai.com)

- **已验证的独立开发者推广平台**: 有案例显示少数派的推荐带来60%下载增长
- 平台主动策展和跨站推广独立应用
- 最佳内容形式: 透明的开发故事+使用教程,而非硬广
- 平台的读者是数字工具重度用户,对效率/生活方式工具有天然兴趣
- 发稿时机: 产品有稳定版本且有真实用户反馈后

#### 3.2 V2EX

- 中国最大的技术社区,用户以开发者和科技从业者为主
- 适合发布技术分享帖("我用XX技术栈做了个约饭App")
- V2EX用户对"自己开发的小工具"容忍度高,对商业推广极度反感
- 发在"分享创造"节点——这是V2EX专门为独立作品展示设置的板块
- 注意: V2EX有严格的反广告规则,帖子必须是真诚分享,不能有营销感

#### 3.3 Product Hunt中文生态

- Product Hunt本身可以发,但中文项目在上面曝光有限
- 中文替代品: 不定期有人在即刻/V2EX/少数派做"中国版PH"式收录
- 更实用的做法是主动联系如"新趣集"等中文产品发现类自媒体

#### 3.4 开源(如果决定开源)

- GitHub开源可以吸引开发者群体的关注和使用
- 附带好处: 技术社区的自发传播(如被收录到awesome-list)
- 权衡: 个人项目是否开源取决于你的偏好,开源不等于更多用户
- 如果开源,可以在README中加入演示链接和使用说明

## 微信生态深度分析

### H5链接分享的完整链路

```
用户点击分享 → 微信JSSDK生成自定义卡片 → 卡片出现在群聊/朋友圈
→ 接收者点击 → 微信内置浏览器打开 → 网页授权(静默/非静默)
→ 获取用户OpenID作为身份标识 → 进入产品流程
```

### 技术必须项

| 需求 | 实现 | 前提条件 |
|------|------|----------|
| 自定义分享卡片 | JSSDK wx.updateAppMessageShareData | 已认证的微信公众号(服务号) |
| 获取用户身份 | 网页授权(静默, snsapi_base) | 已认证的服务号 |
| 微信内置浏览器兼容 | 测试iOS/Android微信内浏览器 | - |
| 基础分享卡片 | Open Graph meta标签 | - (无门槛) |
| HTTPS | 微信要求所有链接为HTTPS | SSL证书 |

### 无服务号的降级方案

如果暂时没有认证服务号(认证需要企业资质和300元/年):

- **Open Graph标签**: 微信内置浏览器会读取`og:title/description/image`生成基础卡片,虽不如JSSDK自定义卡片精美,但可用
- **房间码机制**: 不依赖微信API,完全自主实现
- **用户体验不受影响**: H5本身的功能(创建房间、AI聊天、决策输出)不需要微信API
- **后续升级**: 等验证产品价值后再申请服务号也不迟

### 微信生态的限制与合规

- **链接可能被屏蔽**: 微信会对某些域名做拦截,尤其是包含诱导分享内容的
- **避免诱导分享**: 微信明确禁止"分享后解锁功能""分享得奖励"等机制
- **避免诱导关注**: 不能强制要求关注公众号才能使用
- **内容合规**: 涉及餐饮推荐的内容无敏感风险
- **HTTPS必须**: 微信要求所有网页链接为HTTPS

## 前30天启动计划

### 第1周: 内测(5-10人)
- 目标: 验证核心流程(发起房间→AI对话→决策输出)是否流畅
- 渠道: 直接微信转发给3-5个最亲近的朋友
- 观察: 他们是否能"无说明书"完成一次约饭,在哪里卡住了
- 技术: 确保微信内置浏览器兼容,分享卡片正常显示

### 第2周: 朋友的朋友(10-30人)
- 目标: 验证"操心人→随便党"的传播路径
- 渠道: 邀请内测用户帮你在他们的饭局群里发起一次
- 观察: 随便党是否愿意参与,群聊场景下的使用体验
- 关键指标: 一个房间被使用几次,从发起到决策的平均时间

### 第3周: 小范围社交传播(30-100人)
- 目标: 验证自然传播是否会发生
- 渠道: 不做主动推广,观察现有用户是否自发分享
- 如果发生了自发传播: 记录他们是怎么分享的,分享文案是什么
- 如果没有: 回到产品改进入口体验和分享引导

### 第4周: 根据情况决定
- 如果传播良好: 开始在即刻/朋友圈做有意识的分享
- 如果需要更多反馈: 在V2EX"分享创造"节点发帖
- 如果基本没人用: 优先优化产品而非做推广

## 应规避的渠道(个人项目)

| 渠道 | 原因 |
|------|------|
| 付费广告(微信/抖音/小红书) | 无预算,且个人项目前期投广告ROI极低 |
| KOL/网红合作 | 需要预算(哪怕小KOL也要几千),且"约饭工具"太垂直,泛流量无效 |
| 应用商店( App Store / 应用宝 ) | 这是H5,不是App,不需要 |
| 地推/线下活动 | 成本高,覆盖慢 |
| 企业合作/BD | 个人项目阶段过早,没有商务对接的必要 |
| 小程序(短期内) | 小程序需要审核、备案,增加维护成本;H5链接分享更轻更直接。等H5验证了需求后再考虑小程序化 |

## 关键成功指标(替代GMV/DAU)

对于零预算个人项目,不要追踪传统商业指标:

| 指标 | 含义 |
|------|------|
| 一个房间被打开的次数 | 参与度 |
| 从发起到决策的平均时间 | 效率 |
| 自发分享数(非引导) | 传播力 |
| 回头使用率(同一群本周用了2次以上) | 留存 |
| 操心人-随便党转化率(随便党是否也开始发起房间) | 产品价值验证 |

## 数据缺口

以下信息无法通过公开搜索完整获取,需要通过实际运营来验证:

1. **微信对H5链接的屏蔽率**: 不同域名的拦截概率,需要实际测试
2. **朋友间分享的真实转化率**: "群里发的链接有多少人点"——行业报告给的是平均值,具体到约饭场景需要实测
3. **小红书自然流量的获取难度**: 算法对"AI工具"类内容的推荐力度未知
4. **深圳本地社区的配合效应**: 是否有深圳本地的微信群/公众号愿意自发传播
5. **服务号认证对个人身份的可行性**: 如果没有企业资质,是否有替代方案(如挂靠朋友公司)
