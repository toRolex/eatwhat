## 主要人物画像：「操心人」

**林晓薇，27 岁，深圳南山，产品经理**

> "每次都是我来定餐厅、我来预约、我来排号、我来算人均、我来照顾所有人的忌口——我是朋友还是保姆？"

| 维度 | 详情 |
|------|------|
| 收入 | 月薪 1.5-2.5 万元 |
| 聚餐频率 | 每周 1-3 次（含周末朋友局 + 同事午餐） |
| 常用工具 | 微信、大众点评、小红书、美团 |
| 决策方式 | 先自己搜几家 → 发到群里征求意见 → 反复被否决 → 心力交瘁 |
| 核心痛点 | 承担 100% 的决策心力，得不到感谢，出错还被埋怨 |
| 目标 | 省心。有人（或 AI）替她做掉「协调」环节 |

**行为特征：**
- 群里的「项目经理」——主动发起、主动推进、主动拍板
- 对大众点评评分半信半疑，已学会「翻到第 10 页找真实评价」
- 希望有人分担，但不敢交给「随便党」——上次交出去的结果是肯德基

## 次要人物画像：「随便党」

**张浩，25 岁，深圳福田，后端开发**

> "我都行，你们定。"

| 维度 | 详情 |
|------|------|
| 收入 | 月薪 1.2-2 万元 |
| 聚餐频率 | 每周 1-2 次（被动响应邀请） |
| 常用工具 | 微信、美团 |
| 决策方式 | 不参与协商阶段 → 收到地址 → 导航前往 |
| 核心痛点 | 有隐性偏好，但主动说出太麻烦或有社交压力 |
| 目标 | 快速完成。3 分钟内搞定「告诉我时间地点」 |

**行为特征：**
- 说的「随便」不代表没偏好——是不想承担提议责任
- 有忌口和预算限制，但等决策者主动问才会说
- 决策后如果体验不好，会在心里默默扣分但不说出口

## 决策行为

**典型群体决策流程（现状）：**
```
发起人："周五晚上有空吗？" 
        → 3/5 人回复 → 发起人搜 3-5 家 → 发到群里
        → 2 人否决 → 发起人再搜 → 再发 → 勉强定下来
        → 饭后有人说"其实我想吃XX来着"
耗时：60-120 分钟 + 心力损耗
```

**我们的介入点：**
```
发起人创建房间 → 发链接到群 → 每人各自跟 AI 聊 3 分钟
→ 决策 agent 几秒出方案 → 发到群里确认
耗时：发起人 2 分钟 + 参与者各 3 分钟
```
